# expense-tracker デザイン規約

個人専用・ローカルの支出管理アプリ。**ダークが既定**の「Midnight Ledger」テーマ。
仕様 SoT は [`PROJECT_PLAN.md`](../../../PROJECT_PLAN.md) の §3（パレット）・§6（画面構成）。
本 skill は**どう見せるか**に集中する。色のコード規約（CSS 変数経由・赤の制限）は
[`coding-conventions`](../coding-conventions/SKILL.md) skill と整合させる。

実装前の参照順:
1. UI の最新ブラウザ実装パターン（モーダル・スクロール演出・レイアウト）→ `modern-web-guidance` skill
2. 量産型を避けた質の高い見た目 → `frontend-design` skill
3. 本 skill = このアプリ固有の規律（テーマ・シグナル色・画面構成）

## デザイン原則

- **ダークファースト**。背景は沈め、データを主役にする。`--color-base`(最暗) →
  `--color-surface` → `--color-surface-raised` の3層で奥行きを出す。
- **静かな UI、雄弁な数字**。装飾を足すより、金額・推移を読みやすくする。
- **シグナル色は絞って使う**。色で「意味」を伝え、雰囲気作りに使わない（下記）。
- **3タップで契約登録**（§6 プリセットフロー）など、主要操作の手数を増やさない。

## カラー / テーマ

色は必ず `src/styles/globals.css` の CSS 変数経由（生のカラーコードを直書きしない）。

### 役割
- 面: `--color-base`(背景) / `--color-surface`(カード) / `--color-surface-raised`(浮き要素) / `--color-border`
- 文字: `--color-text-primary` / `--color-text-secondary` / `--color-text-muted`
- 主アクセント: `--color-accent`(teal)。当月総支出など「主役の数字」とプライマリ操作に。
  強調は `--color-accent-glow`、控えめは `--color-accent-dim`。

### シグナル色の規律（重要）
- `--color-danger`(赤) = **値上げ・予算超過 専用**。装飾・通常の強調・削除ボタン以外に使わない。
- `--color-success`(緑) = 減額・予算内。
- `--color-warning`(黄) = 予算接近。`--color-info`(青) = 中立の通知。
- サブスク改定表示: **増額=danger / 減額=success / プラン変更=ニュートラル**。
  例: 「890円 → 1,590円 (+700)」の +700 を赤、減額なら緑。

### カテゴリ別グラフのパレット
円グラフ/凡例はこの順で割り当てる（赤・緑・黄のシグナル色は混ぜない）:
`#2DD4BF #818CF8 #F472B6 #FBBF24 #34D399 #22D3EE`
※定数化して `src/lib`（または theme）に置き、グラフ間で同じ順序を共有する。

## タイポグラフィ / 階層

- 1画面に主役は1つ。ダッシュボードでは**当月総支出を最大・アクセント色**で。
- 金額は等幅感のある数字で桁を揃え、`Intl.NumberFormat('ja-JP')` で `¥1,590` 形式。
- 補助情報（サブスク月額合計・件数・前月比）は `text-secondary`、ラベルは `text-muted`。

## コンポーネント / ライブラリ

- UI: **shadcn/ui**（必要分のみ導入、`src/components/ui/`）。独自実装より優先。
- アイコン: **lucide-react**。サイズ・stroke を揃え、装飾過多にしない。
- グラフ: **Recharts**。配色は上記カテゴリパレット、軸・グリッドは `--color-border`/`text-muted`。
  ツールチップは `--color-surface-raised` 面に。
- フォーム: `src/components/forms/`。日付・金額・カテゴリ・メモ。金額は数値入力＋¥プレフィックス。

## 画面別レイアウト指針（§6 が SoT）

- **ダッシュボード `/`**: 当月総支出(都度＋サブスク)を大きく → サブスク月額合計を併記 →
  カテゴリ別円グラフ → 直近6ヶ月の棒グラフ。上から重要度順。
- **支出一覧 `/expenses`**: 月フィルタを上部固定。都度支出リストと「サブスク(自動)」セクションを
  視覚的に分ける（サブスクは実体でない＝編集導線も別）。
- **サブスク一覧 `/subscriptions`**: 最上部に「月額合計 ¥X,XXX / Y件」。各行=サービス/プラン/金額/
  課金日/編集/解約。直近30日に amount 変更があれば「改定あり」バッジ。
- **プリセット選択 `/subscriptions/new`**: サービス選択 → プラン選択(金額自動入力・編集可) →
  課金日/開始日 → 登録。ステップを視覚化し3タップ感を保つ。
- **改定ログ `/subscriptions/[id]/logs`**: 時系列。増額=赤/減額=緑/プラン変更=中立、差分を明示。
- **設定 `/settings`**: カテゴリ管理(色はパレットから選択) / JSON エクスポート・インポート。

## アクセシビリティ

- ダーク背景でのコントラストを確保（本文 `text-primary`、補助でも `text-secondary` 止まり、
  `text-muted` は非重要情報のみ）。
- 色だけに意味を載せない。増減は色＋符号(+/−)や矢印で二重化する。
- フォーカスリングを残す（キーボード操作）。タップ標的は十分な大きさに。

## Figma / .pen からの落とし込み

- Figma MCP / pencil から受け取った値は**そのまま貼らず**、上記トークン体系に正規化する:
  生のカラー→CSS 変数、px の散らばり→Tailwind spacing スケール、フォント→既定階層。
- デザインデータの赤が「装飾」で使われていたら、本規約（赤=値上げ専用）に合わせて置換する。
